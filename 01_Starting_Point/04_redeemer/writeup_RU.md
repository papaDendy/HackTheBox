# Redeemer — Write-Up

## Summary

Машина уязвима из-за опубликованного Redis-сервиса без механизма аутентификации.

---

## Recon

Полное TCP-сканирование выявило открытый порт 6379, на котором работал Redis.

---

## Enumeration

Подключение к Redis через netcat показало, что сервер отвечает на команды без запроса пароля.

Перечисление ключей позволило обнаружить доступные значения.

---

## Exploitation

С помощью команд Redis (`KEYS *`, `GET <key>`) удалось получить значение ключа, содержащего флаг.

Флаг был успешно извлечён.

---

## Root Cause

- Redis доступен из внешней сети  
- Отсутствует аутентификация  
- Отсутствуют сетевые ограничения  

---

## Mitigation

- Ограничить bind-адрес (127.0.0.1)  
- Включить параметр `requirepass`  
- Настроить firewall  
- Разрешать доступ только доверенным узлам
