# Basic Pentesting: 1 - Write-Up

## Summary

В ходе тестирования была успешно скомпрометирована целевая система.  
Начальный доступ был получен через повторное использование учётных данных, обнаруженных в веб‑приложении.  
Повышение привилегий выполнено за счёт некорректной конфигурации `sudo`, позволявшей запуск `vim` от имени `root` без пароля.

---

## Reconnaissance

Выполнено обнаружение активных хостов в локальной сети и первичное сканирование TCP‑портов целевой системы.

В результате сканирования были обнаружены следующие открытые порты:

- **22/tcp** — SSH
    
- **80/tcp** — HTTP
    

---

## Enumeration

### Service Enumeration

Проведено определение сервисов и их версий на обнаруженных портах:

- **22/tcp** — OpenSSH 7.2p2 (Ubuntu)
    
- **80/tcp** — Apache HTTP Server 2.4.18 (Ubuntu)
    

Аутентификация по SSH без учётных данных невозможна, что исключает данный сервис из вариантов начального доступа на данном этапе.

---

### Web Enumeration

Проведено перечисление директорий веб‑сервера с целью выявления скрытого функционала и утечек информации.

В результате обнаружены следующие директории:

- `/secret/`
    
- `/development/`
    

#### `/secret/`

В директории обнаружена служебная страница с комментариями, содержащими:

- упоминание механизма аутентификации
    
- указание на использование стандартных (default) учётных данных
    

Данная информация указывает на возможное слабое управление учётными записями.

#### `/development/`

В директории размещена служебная страница с описанием используемого стека:

- WordPress 4.7.0
    
- MySQL
    

Также были обнаружены тестовые учётные данные:

- **username:** `dev`
    
- **password:** `devpass123`
    

Полученная информация указывает на возможное повторное использование учётных данных.

---

## Initial Access

Обнаруженные в веб‑приложении учётные данные были использованы для аутентификации по SSH.

В результате был получен интерактивный shell от имени пользователя **dev**, обладающего стандартными пользовательскими привилегиями.

---

## Post‑Exploitation

Было подтверждено, что текущий пользователь не обладает административными правами, что потребовало дальнейшего повышения привилегий для полного контроля над системой.

---

## Privilege Escalation

Проведена проверка `sudo`‑привилегий пользователя `dev`.

В результате выявлена некорректная конфигурация:

- пользователь `dev` может запускать `/usr/bin/vim` от имени `root` без ввода пароля (`NOPASSWD`)
    

Использование данной конфигурационной ошибки позволило выполнить команды от имени `root` и получить полный контроль над системой.

---

## Conclusion

Компрометация системы стала возможной вследствие:

- утечки служебной информации через веб‑интерфейс
    
- повторного использования учётных данных
    
- некорректной конфигурации `sudo`
    

Цепочка уязвимостей позволила эскалировать доступ от неаутентифицированного пользователя до `root`.
